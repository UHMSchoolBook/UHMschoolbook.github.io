Motivation

Whenever a new student starts their journey in a university, their first challenge is to create connectivity with other students. Even older students face this issue in the form of new courses and clubs. The motivation behind this app is to decrease the gap between students on campus while allowing various socializing opportunities designed specifically for the students of UHM.

Goals

The goal of the “UHM-SchoolBook!!” is to improve connectivity in student communities by providing a platform that is specifically designed for students, especially for a particular campus. This app has options to interact as individuals, groups, and members of classes with each other. There are Marketplace, House sharing, and messaging modules to alleviate the college experience of every student.

Usage

  • Login Page

    This page shows up at the start of the app and allows users to login into the app with their credentials. It now uses Firebase for authentication.

  • Registration page

    This page allows new users to create an account in the app so that they can get connected to their peers in the same courses and groups. Now all the data used to create an account is placed on Firebase. Email verification has not been added yet to ensure more testing.

  • Profile Page

    This page shows the profile of the user after login including Bio, Courses student is attending this semester and Clubs they are members of. There is a new button added in the Groups section that alows users to add new groups to their.

  • Courses Description Page

    Clicking on the courses in the profile page now takes student to course description were the description of the course is shown. It also shows the other students taking the course.

  • Groups Description Page

    Clicking on the Groups in the profile page now takes student to Groups description were the description of the Group is shown. It also shows the other students who are members of the group. Now, students can click on leave group to leave the group.

  • Joining Groups

    Now, There is option for students to search and jon new groups. When they click on the add group button on the profile page it opens a search option where they can search for a goup and from the description page they can choose to join the group.

  • Feed page for Courses

    This page shows feeds posted under courses including updates by instructor and other students.

    Feed Posts can now be edited and new posts can be Added.

  • Feed page for Clubs

    This page shows feeds posted under clubs including updates by moderator and other students.

    Feed Posts can now be edited and new posts can be Added.

  • Marketplace page for Items

    The marketplace page displays listing of items put to sale by different students. Now, There is options to massege the poster of the item from the marketplace page. When clicked the send message button, it opens up a chat screen with the poster of the item on the messaging option.

    Items up for sale in marketplace can now be edited and new items can be Added.

  • Housesharing

    Housesharing allows students who want to share their rooms and house with other students for rent sharing the option to do so. Now, There is options to massege the poster of the item from the housesharing page. When clicked the send message button, it opens up a chat screen with the poster of the item on the messaging option.

    Rooms up for sharing can now be edited and new rooms can be Added.

  • Events Page

    An Events Page has been created so that Students can post differents events up there for other students to know.

    New events can be added to the events page that appear throughout the calendars of all the students. The existing events can be edited and deleted by the events creator only

  • Messages

    The messages section now uses Firebase Realtime Messaging API to send and recive messages. Students can search each other form the search bar and send messages.

    Plder Chats also appears in the messages section.

Usability Evaluation Test Report

Initial Usability evaluation for the app has been done. Report available Usability Evaluation Test Design

Top 3 Usability Issues found from the Test were:

  1. Lack of Group search option: (The Issue has been Solved in the latest Version)

    There is no option to search and add groups in the current implementation. Issue Link on Project Board

  2. Missing Message option in Marketplace: (The Issue has been Solved in the latest Version)

    There is no way to message the poster of an item up for sale in the current version. But users can search by the poster's name in the messaging section to contact them. Issue Link on Project Board

  3. Messaging doesn’t appear for new users:

    When a new user signs in into the app for the first time they can't see the messaging option. It fixes on its own when they log out and log in again. Issue Link on Project Board
