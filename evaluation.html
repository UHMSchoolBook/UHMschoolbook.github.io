UHM-SchoolBook Usability Evaluation Test Design

User Goals:

  • Posting Feeds and Updating: Users are able to add new feed posts and update the post for both classes and clubs.
  • Marketplace - Adding and Updating Items: Users are able to add new items in the marketplace and update the item.
  • Messaging other Users: Users can search for other users and send them personal messages successfully.
  • Event Calendar Management: Users can add new events in the event calendar and update or delete the event.
  • House Sharing - Adding and Updating Rooms: Users can add a new room in house sharing and update the room.

Task 1: Add new feed posts and update the post for both classes and clubs

Scenario 1:

You are a student at UH Manoa and willing to post an update about a class you are attending. Use the feed section of the app to post the update and also try updating it.

Scenario 2:

You are a student at UH Manoa and willing to post an update about a club you are a member of. Use the feed section of the app to post the update and also try updating it.

Successful behavior:

Successfully able to create a feed post and edit it with new information.

Task 2: Add new items in the marketplace and update the item.

Scenario:

You are a student at UH Manoa. You have a Table Lamp you no longer need and are willing to sell to their students. Post the Table Lamp as an item for sale on the marketplace section of the app and try updating the price of the Lamp.

Successful behavior:

Successfully able to post an item for sale and change the price or picture of the item.

Task 3: Search for other users and send them personal messages successfully.

Scenario:

You are looking to send a message to a student whom you are not connected through any other means but taking a course together. Try using the message section of the app to search the student using their hawaii.edu mail and sending a message to start a conversation.

Successful behavior:

Successfully able to send messages to other users and can receive messages in realtime.

Task 4: Add new events in the event calendar and update or delete the event.

Scenario:

You are looking to add an event that will appear to all other students in the app. Go to the events section of the app and an event with description. Try updating the event with a new title.

Successful behavior:

Successfully able to post an event into the calendar and can update the event date and description.

Task 5: Add a new room in house sharing and update the room.

Scenario:

You are looking to share the room you are currently living in with a roommate. Go to the house sharing tab in the marketplace. Add the room with address, picture and monthly rent. Try updating the data with new information.

Successful behavior:

Successfully able to post a room up for sharing and can update the address, picture and rent.

User Types:

The UHM-SchoolBook is built to cater the needs of the students studying at UH Manoa. The Important user types for this evaluation will be the students who are currently enrolled in UH Manoa as well as are members of different clubs and groups on campus. Moreover, Students who are looking to share their rooms with other students and also selling products to student communities are also important user types for the evaluation.

Script:

The general script we followed for the testing:

  1. Please create an account on the app with all required information. The photo section uses URL now so use one of your photos uploaded on the web.
  2. When you are done with registration, log in to the app. Check out your profile to ensure everything looks fine, and then move to the Feed page.
  3. Try posting a new feed post with all the information. Check if the post was made as you expected. If you want to update the post, go to the options and try updating it.
  4. Let's say you want to sell an item to the student community. Move to the marketplace page and post an item. Use a URL for the image here also.
  5. The item you posted looks good? Try updating the price of the item.
  6. Let’s try messaging a user now. Move to the messaging page and use the search bar to find one of the developers you want to message. Check if you can see the reply from him.
  7. Try out the event page now. Navigate to the event page and check out recent events posted by other users.
  8. Try posting an event for everyone now. Does it work? Let’s say you made a mistake while creating the event. Try updating it.
  9. Great! Let's delete this event and move to house sharing.
  10. The House Sharing tab is inside the marketplace. Let's say you want to share the current room you are living in with a roommate. Post the room with the address and rent you are expecting to share. Everything looks good? Try updating if not.
  11. Thank you so much for participating in the test. Please fill out this Google form with your experience.

Results:

Top 3 Issues:

  1. Lack of Group search option: There is no option to search and add groups in the current implementation. (The Issue has been Solved in the latest Version) Issue Link on Project Board
  2. Missing Message option in Marketplace: There is no way to message the poster of an item up for sale in the current version. But users can search by the poster's name in the messaging section to contact them. (The Issue has been Solved in the latest Version) Issue Link on Project Board
  3. Messaging doesn’t appear for new users: When a new user signs in into the app for the first time they can't see the messaging option. It fixes on its own when they log out and log in again. Issue Link on Project Board
